# 2026年暑假回国大八项目文档：自动标注与模型训练工作流

本自学/项目文档主要记录如何利用 **X-AnyLabeling** 自动标注工具与 **Roboflow**、**MaixHub** 平台，搭建一套高效的半自动数据标注与 MaixCam 模型部署工作流。

---

## 🛠️ 模块一：X-AnyLabeling 自动标注指南

> **核心场景**：手动拍摄了 1000+ 张图片，一张一张标记会累死。
> **解决方案**：先手动标记 50 张左右的图片，扔去训练一个小模型，然后把这个小模型导入 X-AnyLabeling 用于自动标注，实现“AI自动标 + 人工检查微调”。

### 1. 核心注意事项
1. **模型格式要求**：导入 X-AnyLabeling 的私有模型**需要 `.onnx` 格式**，而不是 `.pt` 格式。
2. **硬件加速（显卡）配置**：
   * 如果从 GitHub 上的 Release 下载 `.exe` 运行，系统将**只会调用 CPU，无法进行硬件加速**。
   * **建议**：使用 `git clone` 源码到本地运行。如果是非 NVIDIA 显卡请无视此条，直接参考 GitHub 官方安装说明进行环境安装。

### 2. 导出数据集格式对比

使用私有模型自动标注完后，导出的数据都是传统框。在导出时可以选择以下不同的数据集格式：

* **i. YOLO HBB 传统框**：用私有模型自动标注完后生成的都是传统框。*（注：目前未研究如何实现类似 Roboflow 同款的涂抹标记）*
* **ii. YOLO OBB**：有向边界框（定向边界框），适用于带旋转角度的目标检测。
* **iii. YOLO Seg 精度高的多边形框（勾边）**：类似 RoboFlow 同款的涂抹标记，高精度，效果好。但标定过程非常麻烦，**除非可以完全实现自动标定，否则不建议使用此格式**。
* **iv. YOLO Pose**：姿态估计/关键点检测数据集格式。
* **v. VOC 检测**：**MaixHub 训练需要的数据集格式**。X-AnyLabeling 导出后会把图片和 `.xml` 标签文件放在同一个文件夹内，后续需要**手动分成两个文件夹**分别存放，即可直接上传至 MaixHub。

### 3. 数据集优化进阶（建议工作流）
* **建议**：在本地完成自动标注与人工校验后，将数据集上传至 **RoboFlow** 平台。
* **数据增强（增加噪音）**：在 RoboFlow 中可以一键添加噪点、运动模糊、白平衡等，以增强模型的鲁棒性（泛化能力）。
* **自动划分数据集**：可以指定比例自动切分训练集（Train）和验证集（Val），系统会自动从标定好的数据中挑选给定比例的数据。
* **格式导出**：处理完成后，可以直接从 RoboFlow 导出 MaixHub 所需的 **PascalVOC** 格式。

## 📷 模块二：MaixHub 平台与 MaixCam 部署规范

> **MaixHub** 是 MaixCam的在线训练部署平台。

### 1. 完整数据闭环工作流

[MaixCam 采集原始图片] -> [上传到 MaixHub] -> [从 MaixHub 下载原始图片] -> [扔给 X-AnyLabeling 自动标记] -> [手动检查与微调] -> [上传到 RoboFlow (自动切分训练/验证集 + 增加噪声)] -> [导出 PascalVOC 格式] -> [重新上传至 MaixHub 进行在线训练] 
*(注：在 RoboFlow 阶段也可以选择导出 YOLO 格式用于本地自主训练)*

### 2. 模型训练硬性规范

#### A. 模型格式与本地训练
* **硬件限制**：MaixCam 使用的是**专有的模型格式**（带 NPU 硬件加速），但是目前官方在线训练平台上**只有 yolov5s**。
* **待研究课题**：后续需要等待并研究如何在本地训练 `.pt` 模型，并将其成功转换为 MaixCam 支持的专有硬件加速模型格式。

#### B. 相机拍摄与分辨率规范（极其重要 ⚠️）
* **i. 必须拍 1:1 的方图！**
  * *原因*：YOLO 格式必须给正方形的图（例如 $640 \times 640$）。如果前期拍摄的不是 1:1 方图，后期训练时图像会被强行拉伸压缩（例如：圆球在视野里会变成椭圆），导致最终训练出来的模型**只会认椭圆**，实机部署时识别率严重下降。
* **ii. 训练分辨率需要为 32 的整数倍！**
  * *原因*：在 MaixHub 平台上排队训练需要等待很久，如果填写的尺寸不是 32 的整数倍，好不容易排到你训练时，系统会因为尺寸不合适**直接打回并报错**，极其浪费排队时间。
